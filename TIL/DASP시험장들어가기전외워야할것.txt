데이터 모델링 기본 원칙
① 커뮤니케이션의 원칙 (Communication Principle)
- 요구 사항은 모든 사람들이 이해할 수 있도록 명확하게 공표됨은 물론 최종 사용자 지향적으로 분명하게 파악되는 수준으로 작성 되어야 한다.
② 모델링의 상세화 원칙 (Gradualrity Principle)
- 데이터의 상세화 정도를 제시하고 조직이 사용하는 정보 구조의 '최소 공통 분모'를 제시해야 한다.
③ 논리적 표현의 원칙(Logical Representation Principle)
- 조직의 데이터에 대한 논리적 측면을 최대한 표현해야 한다. 
- 모델은 물리적 제약조건 없이 비즈니스를 그대로 반영해야 한다.

주제영역에 대한 설명
- 주제 영역은 시스템의 대상이 되는 업무를 명확하게 구분이 가능한 단위 업무로 분리하는 개념이다.
- 주제 영역을 결정할 때는 주제 영역 내부에 존재하게 될 개체들이 높은 결합성을 유지하게 해야한다.
- 주제 영역을 정의함으로 요구 사항 검증 시에 기준으로 활용할 수 있으며, 생산성 향상을 가져온다.
- 데이터 모델과 프로세스 모델은 별개로 진행될 수 있지만 상호 보완적인 위치에 있고 주제 영역은 프로세스 모델링의 기능과 매핑되는 것이 보통이다.
- 데이터 및 업무활동 모델의 품질보증이 용이함.
- 프로젝트 관리가 용이함
- 모델 개발조정이 용이하고 요구사항 검증 시 기준으로 활용 가능함.

논리 데이터 모델링 목적 및 효과
- 해당 비즈니스에 대한 데이터 관점에서의 명확한 이해
- 전사적인 통합 데이터 체계 확립
- 데이터의 일관성 및 정확성 유지를 위한 규칙 도출
- 안정적인 데이터베이스 설계의 토대 마련
- 정보시스템 관계자와의 명확한 의사소통을 위한 수단으로 활용

좋은 데이터 모델의 요소
- 완전성 (Completeness)
- 데이터 재사용 (Data Reusability)
- 중복배제
- 간결성(Elegance)

전사 아키텍처 참조 모델
- 성과 참조 모델 : 정보화 성과의 측정을 위한 항목과 지표 및 방법을 제시한다.
- 업무 참조 모델 : 업무 아키텍처의 기준이 되며, 대상 기관의 사업, 업무 등 전체적으로 분류하고 정의한다.
- 서비스 참조 모델 : 응용아키텍처의 기준이 되며, 응용 서비스의 기능을 분류 및 정의한다.
- 데이터 참조 모델 : 데이터아키텍처의 기준이 되며, 기관 간에 교환되는 주요 데이터 분류 정의를 표준화한다.
- 기술 참조 모델 : 기술아키텍처의 기준이 되며, 정보기술을 분루 및 식별함. 참조 모델 중 가장 많이 사용한다.

* 아크(배타적)관계 관련 설명
- 아크내에 있는 관계는 보통 동일하다.
- 아크내에 있는 관계는 항상 Mandatory이거나 Optional이다.
- 배타적관계는 반드시 하나의 인스턴스에만 속해야 한다.
- 어떤 엔터티는 다수의 배타적 관계를 가질 수 있다. 그러나 지정된 관계는 단 하나의 배타적 관계만 사용해야 한다.

* 논리데이터 모델링 절차
주제영역정의 -> 엔터티정의 -> 관계정의 -> 속성 -> 식별자확정 -> 정규화 -> 이력관리

* 속성검증
1) 원자단위검증 : 사물의 본질을 이루는 고유한 특성이나 성질이 속성이다.
2) 유일값(single value)검증 : 속성에 관리되어야 할 값이 반드시 단 하는 존재 (차량번호, 집주소 등)
3) 추출값(derived valud)검증 : 속성이 원천적인 값인지 다른 속성에 의해 가공되어서 만들어진 값인지를 검증

* 데이터 무결성
ㄱ. 실체무결성
ㄴ. 연쇄작용(트리거)
ㄷ. 참조무결성 : 데이터모델이세 정의된 엔터티간의 관계조건을 유지
ㄹ. 영역(속성) 무결성 
ㅁ. 정규화

- 완전성 : 사용자의 정보 요구 사항이 누락 없이 모두 정의되었는지 확인

* 보안개선의 관리기준
- 불가변성 : 향후 에 재변경되지 않도록 근본적인 개선 방안인지 여부
- 실현가능성 : 현행 기술 수준과 서비스 특징을 고려하여 구현 가능 여부

* 데이터 거버넌스 역량

* 데이터아키텍처 모델링 도구에 대한 설명
- 데이터아키텍처 정보가 수록된 산출물 생성 기능을 제공해야 한다.
- 데이터아키텍처 리포지터리를 보유하고 있어야 한다.
- 데이터 표준 변경 관리 기능을 제공해야 한다.
